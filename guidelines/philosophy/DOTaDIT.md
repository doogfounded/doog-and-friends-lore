# Do One Thing and Do It Well

Try to bring your focus to just one single element rather than trying to juggle everything at once. This means when writing and contributing;

* Dedicate a single issue or pull request to one scene, one character's voice, one thematic thread, or one descriptive passage.
* Avoid "feature creep" when writing drafts (e.g., don't add 10 random subplots when you're trying to figure out the main conflict). Build from fresh when needed meaning starting a PR or issue with a new document, folder, or file instead of bloating one issue, PR, file, or all at once.

The result? Everything is much sharper, like a well-crafted tool that achieves it's goal exceptionally well.
