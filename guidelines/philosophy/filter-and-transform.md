# Filter and Transform (Pipes Mentality)

Try to treat the writing stages as filters:

* Brainstorming → raw idea stream.
* Outlining → structures the flow.
* Drafting → adds flesh.
* Editing → removes noise (extraneous info, like Unix tools avoiding cluttered output).
* Proofreading → polishes the final stream.

It is also permitted to use scripts to make this easier (adverb counting, dialogue, etc) inside the repo.
